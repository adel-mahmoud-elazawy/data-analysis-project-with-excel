# 🚗 Car Sales Interactive Analytics Dashboard (Advanced Excel)


## 📌 Project Overview
This project is a deep-dive analysis of a **Car Sales Dataset** (23K+ records) using **Advanced Excel**. 

As an **AI Engineer**, I previously processed this same dataset using **Python (Pandas/Seaborn)**. In this repository, I showcase how to achieve the same data integrity and extract high-level insights through a dynamic, interactive Dashboard. This cross-tool validation ensures that the data is clean and ready for future **Machine Learning** models.

---

## 🛠️ Technical Workflow (The Engineer's Approach)

### 1. Data Cleaning & ETL (Power Query)
Instead of manual cleaning, I built a robust **ETL pipeline** using **Power Query**:
- **Data Standardization:** Handled missing values and formatted date/currency fields.
- **Data Transformation:** Merged and filtered raw data to ensure consistency with my previous Python analysis.
- **Efficiency:** The pipeline is refreshable; new data can be added without re-doing the work.

### 2. Data Modeling & Analysis
- **Pivot Architecture:** Created structured summaries to analyze sales by body style, engine type, and dealer region.
- **Statistical Validation:** Cross-checked KPIs (Total Sales, Average Price) against my **Python notebook** results to ensure 100% accuracy.

### 3. Interactive Visualization (The Dashboard)
Designed a **Dark-Mode Professional Dashboard** featuring:
- **Dynamic KPIs:** Real-time updates for critical business metrics.
- **Advanced Slicers:** Interactive filtering by Company, Model, and Engine Type.
- **Trend Analysis:** Visualizing sales growth and dealer performance at a glance.

---

## 🧠 Why Excel for an AI Engineer?
While Python is my primary tool for modeling, I used Excel here for:
- **Fast Prototyping:** Quickly visualizing feature correlations.
- **Validation:** Serving as a "Ground Truth" to verify script outputs.
- **Communication:** Bridging the gap between complex code and stakeholder-friendly insights.
